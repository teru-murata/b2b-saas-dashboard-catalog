# Ask Gate Policy v0.3

Ask only when risk requires it:
- Ask at most three questions.
- Do not ask what can be learned from approved local files.
- Record low-risk assumptions and continue.
- Stop with `human_review_required` for unresolved high-risk ambiguity.
- Include a recommended answer for each question.

High-risk topics include public APIs, database schema, authentication, billing, data deletion, deploys, external services, secrets, privacy, production data, artifact retention, protected branch authority, brand identity, accessibility commitments, and third-party asset licensing.
